## ۱. آیا متریال را خوب بررسی کردم؟

### فایل‌های موجود

| فایل | نوع | ابعاد / صفحات | وضعیت بررسی |
|------|-----|----------------|-------------|
| `AIDA Campaign Brochure.pdf` | PDF تصویری (بدون متن قابل استخراج) | ۲ صفحه — افقی A4 تقریبی | ✅ کامل (بصری) |
| `AIDA Campaign Poster.jpg.jpeg` | JPEG | ۷۷۵۰ × ۱۰۹۶۳ px (~۱۳ MB) | ✅ کامل (بصری) |

### نتیجه بررسی

**بله — هر دو فایل به‌طور کامل بررسی شدند.** محتوا، پیام، رنگ‌ها، لوگوها و ساختار اطلاعاتی قابل استفاده برای وب‌سایت است.

### نکات مهم درباره کیفیت فایل‌ها

| موضوع | وضعیت | پیشنهاد |
|-------|--------|---------|
| محتوای بروشور | ✅ مرتب و کامل | متن‌ها را از روی PDF برای وب بازنویسی کنیم (SEO و دسترس‌پذیری) |
| محتوای پوستر | ✅ مرتب و هم‌راستا با بروشور | نسخه بهینه‌شده برای وب لازم است |
| حجم پوستر | ⚠️ ۱۳ MB — برای وب سنگین است | نسخه WebP/JPEG فشرده جداگانه بسازید |
| لوگوها | ⚠️ داخل تصاویر raster هستند | **لوگوهای جدا (SVG/PNG شفاف)** ضروری است |
| فونت | ⚠️ در فایل‌ها embed نشده | فونت فارسی وب (مثلاً Vazirmatn یا IRANSans) انتخاب شود |
| QR / URL | ❌ در متریال دیده نشد | QR و آدرس دامنه نهایی باید بعداً اضافه شود |

**جمع‌بندی:** محتوا و پیام کمپین **مرتب و یکپارche** است؛ برای وب فقط نیاز به **خروجی‌های بهینه‌شده** (لوگو، تصاویر فشرده، متن قابل کپی) داریم.

## ۲. خلاصه پویش آیدا

### نام و شعار

- **نام:** پویش آیدا — آینده دختران ایران
- **شعار اصلی:** «هر انتخاب هوشمندانه، یک سهم از آیدا»
- **تیتر پوستر:** «برای ایجاد ایمنی علیه HPV — آگاهی، انتخاب هوشمندانه و مشارکت اجتماعی»
- **همکاری:** پویش آیدا با همراهی **گاردیسان**

### هدف پویش

افزایش آگاهی و فراهم‌سازی دسترسی عادلانه به واکسیناسیون ویروس پاپیلومای انسانی (HPV).

### مدل اثر اجتماعی (مهم‌ترین پیام عملی)

> **به ازای هر ۱۰ نفر** که در پویش مشارکت می‌کنند، **۱ واکسیناسیون رایگان** برای فردی در نقاط کم‌برخوردار ایران فراهم می‌شود.

### «انتخاب هوشمندانه» چیست؟ (۴ گام)

1. آگاهی درباره HPV و پیشگیری
2. مشورت با پزشک یا داروساز
3. اقدام از مسیرهای معتبر
4. مشارکت در سهم واکسیناسیون رایگان

### مسیر تصمیم‌گیری مسئولانه (۳ گام — از بروشور)

1. **آگاهی**
2. **مشورت**
3. **اقدام مجاز**

### محصول: گاردیسان

- واکسن HPV چهارظرفیتی (Recombinant)
- پوشش serotypes: **۶، ۱۱، ۱۶، ۱۸**
- گروه هدف: نوجوانان، جوانان و بزرگسالان جوان (دختران و پسران)
- برنامه دوز در مطالعه: دوز ۱ (ماه ۰) → دوز ۲ (ماه ۲) → دوز ۳ (ماه ۶)
- ایمن‌زایی: sero-conversion ~۹۷٪ برای هر چهار type

### شرکا / لوگوها

1. **گاردیسان** (GARDISUN)
2. **بیوسان فارمد** (BIOSUN PHARMED)
3. **مؤسسه خیریه کنترل سرطان ایران — MACSA**

### پیام‌های الزام‌آور (باید در سایت باشد)

- مطالب صرفاً آگاهی‌رسانی عمومی است
- تصمیم واکسیناسیون باید **آگاهانه** و **پس از مشورت با پزشک یا داروساز** باشد
- خرید فقط از **منابع معتبر** با رعایت **زنجیره سرد**

### قانون مشارکت در پویش (طبق درخواست شما)

> زمان مشارکت در پویش **زمانی ثبت می‌شود** که فرد واکسن گاردیسان خریداری‌شده از داروخانه را در وب‌سایت **ثبت کند** (عکس + سریال + اطلاعات فرم).

## ۷. صفحه ثبت مشارکت (فرم)

### ۷.۱ — فیلدهای فرم

| # | فیلد | نوع | الزامی | توضیح |
|---|------|-----|--------|--------|
| 1 | نام | `text` | ✅ | حداقل ۲ کاراکتر فارسی |
| 2 | نام خانوادگی | `text` | ✅ | حداقل ۲ کاراکتر |
| 3 | شماره تماس | `tel` | ✅ | موبایل ایران: `09XXXXXXXXX` |
| 4 | جنسیت | `select` | ✅ | زن / مرد |
| 5 | سن | `number` | ✅ | محدوده ۹–۵۰ (قابل تنظیم) |
| 6 | تاریخ خرید/دریافت واکسن | `date` | ✅ | تاریخ شمسی یا میلادی (پیشنهاد: date picker شمسی ساده) |
| 7 | شماره سریال محصول | `text` | ✅ | از روی جعبه/برچسب واکسن |
| 8 | عکس محصول | `file` | ✅ | JPEG/PNG/WebP — حداکثر ۵ MB |
| 9 | پذیرش شرایط | `checkbox` | ✅ | «اطلاعات صحیح است / با پزشک مشورت کرده‌ام» |
| 10 | (مخفی) UTMها | `hidden` | — | از sessionStorage پر می‌شود |

### ۷.۲ — فیلدهای اختیاری (پیشنهاد برای فاز ۲)

| فیلد | دلیل |
|------|------|
| استان / شهر | آمار جغرافیایی |
| نام داروخانه | اعتبارسنجی |
| شماره دوز (۱، ۲ یا ۳) | پیگیری برنامه واکسination |

> **فاز ۱:** فقط فیلدهای الزامی بالا. ساده نگه می‌داریم.

### ۷.۳ — UX فرم

- **Progress:** فرم یک‌مرحله‌ای (نه wizard چندمرحله‌ای — ساده‌تر)
- **اعتبارسنجی:** سمت کلاینت (JS) قبل از ارسال
- **Loading:** دکمه «در حال ارسال...» + غیرفعال شدن دوباره‌کلیک
- **خطا:** پیام فارسی واضح (شبکه، حجم عکس، فیلد خالی)
- **راهنمای عکس:** تصویر نمونه ۴۰۰px «این قسمت از جعبه باید readable باشد»

### ۷.۴ — ارسال به n8n

**روش:** `POST` به URL webhook n8n  
**Content-Type:** `multipart/form-data` (به‌خاطر عکس)

#### Payload (فیلدها)

```
first_name
last_name
phone
gender
age
purchase_date        (ISO 8601: 2026-06-13)
serial_number
product_photo        (فایل باینری)
utm_source
utm_medium
utm_campaign
utm_term
utm_content
page_url             (URL صفحه ثبت هنگام ارسال)
submitted_at         (timestamp ISO — سمت JS)
user_agent
```

#### پاسخ مورد انتظار از n8n (Webhook Response)

```json
{
  "success": true,
  "tracking_code": "AIDA-7K2M9P",
  "message": "با تشکر از مشارکت شما در پویش آیدا."
}
```

#### جریان (Flow)

```
کاربر پر می‌کند → JS validate → POST webhook n8n
                                      ↓
                              n8n: ذخیره + تولید کد
                                      ↓
                              Webhook Response JSON
                                      ↓
                    JS: redirect → thank-you.html?ref=AIDA-7K2M9P
```

> جزئیات فنی n8n در فایل [`03-webhook-contract.md`](./03-webhook-contract.md)

## ۸. کد رهگیری

### پیشنهاد فرمت

```
AIDA-XXXXXX
```

| جزء | توضیح |
|-----|--------|
| `AIDA` | پیشوند ثابت برند |
| `-` | جداکننده |
| `XXXXXX` | ۶ کاراکتر alphanumeric (بدون O/0/I/1 برای خوانایی) |

**مثال:** `AIDA-7K2M9P` — طول کل: **۱۱ کاراکتر**

### چرا این فرمت؟

- کوتاه enough برای dictation تلفنی
- قابل تشخیص visually
- فضای ۳۲^۶ ≈ ۱ میلیارد ترکیب (کاراکترها: `23456789ABCDEFGHJKLMNPQRSTUVWXYZ`)

### تولید کد

- **در n8n** (نه در frontend) — برای یکتا بودن
- Node: `Code` → random 6 chars → check duplicate (اختیاری)

### نمایش در thank-you

- فونت بزرگ monospace
- دکمه «کپی کد»
- متن: «این کد را نگه دارید»

| `utm_campaign` | `aida2026` | نام کمپین |
| `utm_term` | `hpv` | کلیدواژه |
| `utm_content` | `poster_v1` | نسخه creative |

### پیاده‌سازی JS (`utm.js`)

1. هنگام ورود به **هر صفحه**: خواندن query string
2. اگر UTM وجود داشت → ذخیره در `sessionStorage`
3. هنگام submit فرم → attach به FormData
4. UTM اول session برنده است (first-touch)

### مثال لینک‌های کمپین

```
https://aida.example.com/?utm_source=instagram&utm_medium=story&utm_campaign=aida2026
https://aida.example.com/register?utm_source=pharmacy&utm_medium=qr&utm_campaign=aida2026
```
